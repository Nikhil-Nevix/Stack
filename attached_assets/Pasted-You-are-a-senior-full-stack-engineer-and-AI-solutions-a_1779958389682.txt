SECTION 2 — SYSTEM OVERVIEW
════════════════════════════════════════════════════════════

STACK automates IT service desk ticket resolution by:
- Accepting tickets via Web Dashboard, Google Chat, Teams
- Running AI triage using Azure OpenAI GPT-4o + LangChain
- Retrieving SOPs via RAG pipeline (pgvector)
- Auto-resolving tickets via Microsoft Graph API / PowerShell
- Escalating to human agents when AI confidence is low
- Logging everything to PostgreSQL + Elasticsearch
- Displaying real-time dashboards, logs, reports, ROI metrics

Four use cases:
  UC-01: SharePoint Tickets (Access + Admin)
  UC-02: License Requests (BlueBeam, Adobe, O365)
  UC-03: DL Update Requests (O365)
  UC-04: Basic Windows Troubleshooting

Three user-facing channels:
  1. React.js Web Dashboard
  2. Google Chat Bot
  3. Microsoft Teams Bot

SECTION 3 — RBAC SPECIFICATION
════════════════════════════════════════════════════════════

Implement full Role-Based Access Control with 3 roles:
  admin    → full system access
  agent    → ticket handling + limited config
  user     → self-service only (raise + track own tickets)

3.1 ROLE DEFINITIONS

ADMIN (admin@jadeglobal.com and any role=admin account):
  - Full access to ALL pages and features
  - Can view ALL tickets (not just own)
  - Can configure confidence thresholds per use case
  - Can configure SLA settings
  - Can manage agent groups and assignment modes
  - Can add/remove/edit agents
  - Can upload/edit/delete SOPs
  - Can view full audit logs and API call logs
  - Can view ROI dashboard and change cost settings
  - Can trigger AI resolution on any ticket
  - Can manually escalate or close any ticket
  - Can export all reports
  - Can manage user accounts
  - Sees Admin sidebar menu item
  - Sees SOP Manager sidebar menu item
  - Sees all columns in ticket table including AI internals

AGENT (role=agent):
  - Can view tickets assigned to their groups
  - Can add notes to assigned tickets
  - Can resolve/close assigned tickets
  - Can view reports (read-only, no export)
  - Can view audit logs for their tickets only
  - Cannot access Admin configuration page
  - Cannot change confidence thresholds or SLA configs
  - Cannot manage other agents or groups
  - Can view ROI dashboard (read-only)
  - Cannot upload or edit SOPs
  - Does NOT see Admin or SOP Manager in sidebar

USER (role=user — end users like testuser@jadeglobal.com):
  - Can ONLY see their own tickets
  - Can raise new tickets via web dashboard
  - Can view status and notes on own tickets
  - Can add comments to own tickets
  - Cannot see other users' tickets
  - Cannot see any admin configuration
  - Cannot see audit logs, API logs, PowerShell logs
  - Cannot see ROI dashboard
  - Cannot see agent performance reports
  - Sees a simplified dashboard:
    - My Open Tickets count
    - My Resolved Tickets count
    - Raise New Ticket button (prominent)
    - My recent tickets list
  - Simplified sidebar:
    🏠 My Dashboard
    🎫 My Tickets
    ➕ Raise Ticket
    👤 My Profile

3.2 RBAC IMPLEMENTATION

Backend:
  - JWT payload includes: user_id, email, role, name
  - FastAPI dependency: get_current_user() → decodes JWT
  - Role guards:
    require_admin() → role must be 'admin'
    require_agent_or_admin() → role in ['admin','agent']
    require_any_auth() → any valid JWT
  - Data filtering:
    Users: all queries filter by user_id = current_user.id
    Agents: queries filter by assigned group use cases
    Admins: no filtering, see everything

Frontend:
  - After login, decode JWT to get role
  - Store role in Redux auth slice
  - Route protection:
    /admin/* → redirect to /dashboard if not admin
    /sops/* → redirect if not admin
    /reports/export → hide if user role
    /logs/audit → hide entirely for users
  - Render sidebar based on role (3 different sidebar configs)
  - Hide columns, buttons, and sections based on role
  - Show "Not authorized" component if URL accessed directly

3.3 LOGIN PAGE SPECIFICATION

  Full-screen login page (no sidebar):
  - STACK logo + "Service Desk AI Solution" centered
  - "by Jade Global" in orange below logo
  - Email field with @jadeglobal.com validation
    (show error inline if domain not @jadeglobal.com)
  - Password field with show/hide toggle
  - "Sign In" button (navy background, orange hover)
  - Below button: "Sign in with Google Workspace" button
    (Google SSO — only allow @jadeglobal.com accounts)
  - Error states:
    Wrong password: "Invalid credentials. Please try again."
    Wrong domain: "Access restricted to Jade Global employees."
    Account inactive: "Your account has been deactivated."
  - After login: redirect based on role:
    admin/agent → /dashboard (full dashboard)
    user → /my-dashboard (simplified user dashboard)

SECTION 5 — DATABASE SCHEMA (PostgreSQL + pgvector)
════════════════════════════════════════════════════════════

Database: PostgreSQL 16 with pgvector extension
ORM: SQLAlchemy async with Alembic migrations

ENUMS:
  use_case: sharepoint_access, sharepoint_admin,
    license_bluebeam, license_adobe, license_o365,
    dl_update, windows_troubleshooting
  status: open, in_progress, auto_resolved, escalated, closed
  priority: low, medium, high, urgent
  source: freshservice, web_dashboard, google_chat,
          microsoft_teams
  sla_status: safe, at_risk, breached
  resolution_type: auto, manual
  assignment_mode: round_robin, first_available, priority
  actor_type: user, ai, agent, system
  note_type: ai_context, resolution_summary,
             human_note, system_note
  execution_status: success, failed, timeout, partial
  decision: auto_resolve, review_after, escalate
  user_role: admin, agent, user
  platform: google_chat, microsoft_teams, web_dashboard

TABLE: users
  user_id UUID PK DEFAULT gen_random_uuid()
  email VARCHAR UNIQUE NOT NULL
    CONSTRAINT email_domain CHECK (email LIKE '%@jadeglobal.com')
  full_name VARCHAR NOT NULL
  role user_role ENUM DEFAULT 'user'
  department VARCHAR
  device_name VARCHAR
  device_ip VARCHAR
  google_workspace_id VARCHAR NULLABLE
  teams_user_id VARCHAR NULLABLE
  freshservice_requester_id VARCHAR NULLABLE
  is_active BOOLEAN DEFAULT TRUE
  password_hash VARCHAR NOT NULL
  last_login TIMESTAMP NULLABLE
  created_at TIMESTAMP DEFAULT NOW()
  updated_at TIMESTAMP

TABLE: agents
  agent_id UUID PK DEFAULT gen_random_uuid()
  user_id UUID FK → users.user_id UNIQUE
  freshservice_agent_id VARCHAR
  is_active BOOLEAN DEFAULT TRUE
  created_at TIMESTAMP DEFAULT NOW()
  (agent = user with role='agent' or role='admin'
   this table extends users for agent-specific data)

TABLE: tickets
  ticket_id UUID PK DEFAULT gen_random_uuid()
  freshservice_ticket_id VARCHAR UNIQUE NULLABLE
  title VARCHAR NOT NULL
  description TEXT
  use_case use_case ENUM NOT NULL
  status status ENUM DEFAULT 'open'
  priority priority ENUM DEFAULT 'medium'
  sla_deadline TIMESTAMP
  sla_status sla_status ENUM DEFAULT 'safe'
  sla_breach_predicted BOOLEAN DEFAULT FALSE
  created_at TIMESTAMP DEFAULT NOW()
  updated_at TIMESTAMP
  closed_at TIMESTAMP NULLABLE
  source source ENUM NOT NULL
  user_id UUID FK → users.user_id
  assigned_agent_id UUID FK → users.user_id NULLABLE
  resolution_type resolution_type ENUM NULLABLE

TABLE: ticket_notes
  note_id UUID PK
  ticket_id UUID FK → tickets.ticket_id CASCADE DELETE
  note_type note_type ENUM
  content TEXT NOT NULL
  created_by UUID FK → users.user_id
  is_internal BOOLEAN DEFAULT FALSE
    (internal=TRUE: only visible to agents/admins)
  created_at TIMESTAMP DEFAULT NOW()

TABLE: ticket_attachments
  attachment_id UUID PK
  ticket_id UUID FK → tickets.ticket_id CASCADE DELETE
  file_name VARCHAR
  file_path VARCHAR
  file_size INTEGER
  uploaded_by UUID FK → users.user_id
  uploaded_at TIMESTAMP DEFAULT NOW()

TABLE: ai_resolutions
  resolution_id UUID PK
  ticket_id UUID FK → tickets.ticket_id
  intent_detected VARCHAR
  root_cause TEXT
  sop_matched UUID FK → sops.sop_id NULLABLE
  confidence_score FLOAT
  intent_clarity_score FLOAT
  sop_match_score FLOAT
  historical_success_score FLOAT
  input_completeness_score FLOAT
  decision decision ENUM
  resolution_steps JSONB
  execution_status execution_status ENUM
  execution_output TEXT
  time_taken_seconds INTEGER
  created_at TIMESTAMP DEFAULT NOW()

TABLE: sops
  sop_id UUID PK
  title VARCHAR NOT NULL
  use_case use_case ENUM NOT NULL
  content TEXT NOT NULL
  embedding_vector VECTOR(1536)
  version VARCHAR DEFAULT '1.0'
  is_active BOOLEAN DEFAULT TRUE
  created_by UUID FK → users.user_id NULLABLE
  created_at TIMESTAMP DEFAULT NOW()
  updated_at TIMESTAMP

TABLE: powershell_executions
  execution_id UUID PK
  ticket_id UUID FK → tickets.ticket_id
  resolution_id UUID FK → ai_resolutions.resolution_id
  device_name VARCHAR
  device_ip VARCHAR
  script_name VARCHAR
  script_content TEXT
  execution_status execution_status ENUM
  output_log TEXT
  executed_at TIMESTAMP DEFAULT NOW()
  duration_seconds INTEGER

TABLE: agent_groups
  group_id UUID PK
  group_name VARCHAR NOT NULL
  use_case use_case ENUM NOT NULL
  assignment_mode assignment_mode ENUM DEFAULT 'round_robin'
  freshservice_group_id VARCHAR NULLABLE
  created_at TIMESTAMP DEFAULT NOW()

TABLE: agent_group_members
  member_id UUID PK
  group_id UUID FK → agent_groups.group_id
  user_id UUID FK → users.user_id
  priority_order INTEGER DEFAULT 1
  joined_at TIMESTAMP DEFAULT NOW()

TABLE: audit_logs
  log_id UUID PK
  ticket_id UUID FK → tickets.ticket_id NULLABLE
  event_type VARCHAR NOT NULL
  actor_id UUID FK → users.user_id NULLABLE
  actor_type actor_type ENUM
  details JSONB
  ip_address VARCHAR
  platform platform ENUM NULLABLE
  created_at TIMESTAMP DEFAULT NOW()

TABLE: api_call_logs
  api_log_id UUID PK
  ticket_id UUID FK → tickets.ticket_id NULLABLE
  api_name VARCHAR NOT NULL
  endpoint VARCHAR
  method VARCHAR
  request_payload JSONB
  response_status INTEGER
  response_payload JSONB
  duration_ms INTEGER
  called_at TIMESTAMP DEFAULT NOW()

TABLE: chat_sessions
  session_id UUID PK
  user_id UUID FK → users.user_id
  ticket_id UUID FK → tickets.ticket_id NULLABLE
  platform platform ENUM NOT NULL
  messages JSONB
  current_flow VARCHAR NULLABLE
  current_step INTEGER DEFAULT 0
  collected_data JSONB
  session_start TIMESTAMP DEFAULT NOW()
  session_end TIMESTAMP NULLABLE

TABLE: confidence_thresholds
  threshold_id UUID PK
  use_case use_case ENUM UNIQUE NOT NULL
  auto_resolve_min FLOAT DEFAULT 85.0
  review_after_min FLOAT DEFAULT 60.0
  updated_by UUID FK → users.user_id
  updated_at TIMESTAMP DEFAULT NOW()

TABLE: sla_configs
  sla_id UUID PK
  use_case use_case ENUM NOT NULL
  priority priority ENUM NOT NULL
  resolution_hours INTEGER NOT NULL
  warning_threshold_percent FLOAT DEFAULT 75.0
  updated_at TIMESTAMP DEFAULT NOW()
  UNIQUE(use_case, priority)

TABLE: roi_metrics
  metric_id UUID PK
  period_start DATE NOT NULL
  period_end DATE NOT NULL
  total_tickets INTEGER DEFAULT 0
  auto_resolved_count INTEGER DEFAULT 0
  manual_resolved_count INTEGER DEFAULT 0
  avg_auto_resolution_mins FLOAT
  avg_manual_resolution_mins FLOAT
  hours_saved FLOAT
  cost_saved FLOAT
  agent_hourly_cost FLOAT DEFAULT 500.0
  calculated_at TIMESTAMP DEFAULT NOW()

## 12. COMMON ISSUES & TROUBLESHOOTING
  Issue: "pgvector extension not found"
  Fix: Run: CREATE EXTENSION IF NOT EXISTS vector;
       In the stackdb database

  Issue: "Azure OpenAI 401 error"
  Fix: Check AZURE_OPENAI_API_KEY in .env
       Verify deployment name matches AZURE_OPENAI_DEPLOYMENT

  Issue: "WinRM connection refused"
  Fix: Enable WinRM on target: winrm quickconfig
       Open port 5985 (HTTP) or 5986 (HTTPS) in firewall
       Set-Item WSMan:\localhost\Client\TrustedHosts -Value "*"

  Issue: "Google Chat webhook 401"
  Fix: Verify service account JSON file path in .env
       Ensure bot is added to Google Chat space

  Issue: "Teams bot not responding"
  Fix: Verify TEAMS_APP_ID and TEAMS_APP_PASSWORD in .env
       Check bot endpoint registered in Azure Bot Service
       Ensure /webhook/teams is publicly accessible

  Issue: "Celery tasks not running"
  Fix: Check Redis connection: redis-cli ping
       Restart celery worker: docker-compose restart celery_worker
       Check beat schedule: docker-compose logs celery_beat
